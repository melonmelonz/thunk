# thunk M-F: The Offline Web GUI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A gorgeous, fully offline, static web rendering of the whole course - reader, checks with
immediate feedback, the simulated panel, and the bus trace - generated from the same content
pipeline, with zero external assets, WCAG-minded semantics, and the minimal instrument feel the
UI/UX research specifies. `thunk web --out <dir>` writes the site; opening `index.html` from disk
works with no server; `thunk serve` (127.0.0.1 only) is a convenience.

**Architecture:** Per `docs/research/ui-ux-research.md` (binding): plain semantic HTML generated by
Rust, NOT a terminal-grid wasm app. New crate `thunk-web` = pure functions from domain values to
HTML/CSS strings (fully unit-testable), plus a thin writer. A ~90-line markdown renderer covers the
constrained lesson dialect (guaranteed by thunk-content's validation suite). Checks grade
client-side with one small hand-written `check.js` (no framework, no fetch, no external anything;
answer data lives in `data-` attributes - the same answers the binary embeds). The panel renders as
run-length-compressed SVG rects from the decoded `Ili9341` framebuffer; the trace renders
`annotate()` rows as a semantic table. Theming: ~20 CSS custom properties, two themes (dark
"gantry" default, light "datasheet") via `prefers-color-scheme` + a JS-free `<meta>`-less toggle
(checkbox hack or a 3-line inline script; keep it honest and tiny). Fonts: system stacks for now
(research's IBM Plex embedding needs font files this offline sandbox cannot fetch; note it as a
follow-up in the roadmap, do NOT reference any external font).

**Tech Stack:** Rust 1.88, ZERO new dependencies (std only in thunk-web; thunk-core/content/sim as
path deps). Hand-written CSS/JS as `include_str!` assets inside thunk-web.

## File Structure

```
thunk-web/
  Cargo.toml               # lib; deps: thunk-core, thunk-content, thunk-sim (path)
  src/lib.rs               # exports + Site::generate() -> Vec<(PathBuf, String)>
  src/markdown.rs          # the constrained-dialect renderer (pure)
  src/page.rs              # layout shell, nav, escaping helpers (pure)
  src/course.rs            # index / module / lesson pages incl. check widgets (pure)
  src/sim.rs               # panel SVG + trace table page (pure)
  assets/thunk.css         # tokens + layout + themes (hand-written)
  assets/check.js          # check grading + aria-live feedback (hand-written, ~60 lines)
thunk-cli/                 # `thunk web --out DIR` + `thunk serve` (std TcpListener, 127.0.0.1)
  src/main.rs
.github/workflows/ci.yml   # + web job: generate site, assert zero external URLs
```

Design tokens (from the research; implement exactly): `--bg`, `--bg-raised`, `--fg`, `--fg-muted`,
`--accent`, `--accent-2`, `--ok`, `--err`, `--border`, `--code-bg`, `--trace-hi`, plus type scale
`--text-s/m/l`, `--measure` (65ch), `--leading` (1.65), `--radius`, `--gap-s/m/l`, `--mono`,
`--sans` (system stacks). Dark gantry default; light datasheet under `prefers-color-scheme: light`.

    fn headings_paragraphs_and_bold() {
        let html = render("# The Machine\n\nA computer is a **machine** that follows instructions.\n");
        assert!(html.contains("<h1>The Machine</h1>"));
        assert!(html.contains("<p>A computer is a <strong>machine</strong> that follows instructions.</p>"));

        let html = render("```\nlet a = <b> & 'c';\n```\n");

    }

    #[test]
    fn inline_code_and_lists() {
        let html = render("- **bit** — a digit\n- **byte** — eight bits\n\nUse `mmap` here.\n1. first\n2. second\n");
        assert!(html.contains("<ul>"));
        assert!(html.contains("<li><strong>bit</strong> — a digit</li>"));
        assert!(html.contains("<code>mmap</code>"));
        assert!(html.contains("<ol>"));
        assert!(html.contains("<li>first</li>"));
    }

    #[test]
    fn html_in_prose_is_escaped() {
        let html = render("a < b & c > d\n");
        assert!(html.contains("a &lt; b &amp; c &gt; d"));
        assert!(!html.contains("<script"));
    }

    #[test]
    fn every_real_lesson_renders_without_panicking_and_balanced() {
        for m in thunk_content::Curriculum::all() {
            for l in &m.lessons {
                let html = render(&l.body);
                assert_eq!(html.matches("<pre>").count(), html.matches("</pre>").count(), "{}", l.id.0);
                assert_eq!(html.matches("<ul>").count(), html.matches("</ul>").count(), "{}", l.id.0);
                assert!(!html.contains("```"), "unrendered fence in {}", l.id.0);
            }
        }
    }
}
